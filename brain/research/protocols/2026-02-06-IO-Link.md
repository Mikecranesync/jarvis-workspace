## 1. Overview

IO-Link is a standardized, point-to-point communication protocol for smart sensors and actuators (IEC 61131-9). It's the "USB of industrial sensors" - bidirectional, plug-and-play, with automatic device identification.

**Key Features:**
- Bidirectional communication (read parameters, diagnostics, not just process data)
- Device identification (automatic sensor recognition)
- Parameterization (configure sensors remotely)
- Diagnostic data (health, errors, warnings)
- Standardized by IO-Link Consortium

**Who Uses It:**
- Growing adoption across all industries
- Major sensor manufacturers (IFM, Balluff, Sick, Turck, Pepperl+Fuchs)
- Especially popular in automotive, packaging, food/beverage

## 2. Physical Layer

| Spec | Value |
|------|-------|
| Wiring | Standard 3-wire (24V, GND, Data) |
| Connector | M12 (typically A-coded) |
| Max Distance | 20 meters |
| Data Rates | COM1: 4.8 kbaud, COM2: 38.4 kbaud, COM3: 230.4 kbaud |
| Power | 24V DC via same cable |

**Key Advantage:** Uses existing 3-wire sensor cables - no special wiring needed.

---

## 3. Data Capabilities

- **Process Data:** Cyclic sensor readings (up to 32 bytes)
- **Service Data:** Acyclic parameter read/write
- **Events:** Alarms, warnings, notifications
- **IODD Files:** Device description files (like GSD for PROFINET)
