# Procedure Library

Documented procedures that have proven successful. Reference these when tackling similar tasks.

## Structure

Each procedure file follows this format:
```markdown
# Procedure: [Name]

## When to Use
- Trigger conditions

## Steps
1. Step one
2. Step two
...

## Success Criteria
- How to know it worked

## Common Pitfalls
- What to avoid

## Examples
- Past successful uses
```

## Categories

- `research/` — Research and information gathering
- `coding/` — Code changes and fixes
- `communication/` — Messaging and notifications
- `automation/` — Setting up cron jobs and workflows
- `debugging/` — Troubleshooting issues
